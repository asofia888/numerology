# 数秘術鑑定 | Numerology Reading

生年月日とお名前(ローマ字)から、ピタゴラス式数秘術に基づいてプロの鑑定書スタイルの読み解きを行う単一HTMLアプリです。サーバー・ビルド不要、`index.html` 1ファイルだけで動作します。

## 機能

**個人鑑定**
- 6つのコアナンバー(ライフパス/ディスティニー/ソウル/パーソナリティ/バースデー/マチュリティ)
- 全12ナンバー(1〜9・11・22・33)× 基本性質・才能・成長のテーマ・恋愛・仕事・開運の鍵
- マスターナンバー(11・22・33)は還元せず保持
- カルミックナンバー(13・14・16・19)の検出と解説
- パーソナルイヤー/パーソナルマンス/来年の流れ(9年サイクル)
- 「計算の内訳」で文字ごとの換算表と還元過程を表示

**相性診断**
- ライフパス45通りすべて書き下ろしの相性鑑定(調和度・ご縁のかたち・アドバイス)
- 魂の共鳴(ソウル)/外から見たお二人(パーソナリティ)
- 二人のパーソナルイヤー比較

**共通**
- 鑑定書の印刷・PDF保存(白背景に自動反転)
- スマートフォン最適化/レスポンシブ対応
- スマートフォン表示では画面右上のボタンで文字サイズを切替可能(小・標準・大。選択は端末内に記憶)
- ライフパス計算方式の切替(一括合算方式/桁別還元方式)
- 全角英字・マクロン付きローマ字(Ō・Ū など)の自動補正
- 選択した年・月に合わせて「日」の選択肢を自動調整
- 入力された情報は端末内でのみ計算(外部への送信・保存は一切なし)

## GitHub Pages での公開手順

1. GitHubで新しいリポジトリを作成(例: `numerology-reading`)
2. このフォルダ内のファイル(`index.html`・`og-image.png`・`README.md`)をアップロードしてコミット
3. リポジトリの **Settings → Pages** を開く
4. **Source** で「Deploy from a branch」を選び、Branch を `main` / `(root)` に設定して Save
5. 数分後、`https://ユーザー名.github.io/リポジトリ名/` で公開されます
6. OGP タグ(SNSシェア用カード)は `https://asofia888.github.io/numerology/` 向けに設定済みです。独自ドメインなど別のURLで公開する場合のみ、`index.html` 内の `og:url`・`og:image` の2行を書き換えてください

## 技術メモ

- 依存ライブラリなし(Google Fonts の Shippori Mincho / Cormorant Garamond のみ読み込み)
- 計算方式: ピタゴラス式(A=1〜Z=8)。母音は A・E・I・O・U、Y は子音扱い
- ライフパスは「一括合算方式(生年月日の全桁を合計)」と「桁別還元方式(年・月・日を個別に還元して合算)」を画面上で切替可能(初期値: 一括合算)。数秘術は流派により計算方式が異なり、マスターナンバーの出方が変わることがあるため両対応
- パーソナルイヤーは 1月1日切り替えで算出
- 名前入力はヘボン式ローマ字を想定(全角英字・マクロン付き文字「Ō」などは自動で半角・基本ラテン文字に補正)
- OGP・Twitterカード対応(シェア用画像 `og-image.png` 同梱。公開後に `og:url`・`og:image` のURL書き換えが必要)
