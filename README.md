# 数秘術鑑定 | Numerology Reading

生年月日とお名前(ローマ字)から、ピタゴラス式数秘術に基づいてプロの鑑定書スタイルの読み解きを行う単一HTMLアプリです。サーバー・ビルド不要、`index.html` 1ファイルだけで動作します。

公開中のサイト: https://numerology.matrix.jp/(さくらインターネット/運営:Wind Bell)

## 機能

**個人鑑定**
- 6つのコアナンバー(ライフパス/ディスティニー/ソウル/パーソナリティ/バースデー/マチュリティ)
- 全12ナンバー(1〜9・11・22・33)× 基本性質・才能・成長のテーマ・恋愛・仕事・開運の鍵
- マスターナンバー(11・22・33)は還元せず保持
- カルミックナンバー(13・14・16・19)の検出と解説
- パーソナルイヤー/パーソナルマンス/パーソナルデー/来年の流れ(9年サイクル)
- これから12ヶ月の運気ごよみ(各月のパーソナルマンスを一覧表示。今月を自動ハイライト)
- 上級鑑定:ピナクル&チャレンジ(人生を4つの季節に分けた長期テーマと課題。現在の季節を自動ハイライト)
- 上級鑑定:人生の三大周期(形成期・生産期・収穫期のサイクルナンバー。周期の変わり目は27歳前後・54歳前後のパーソナルイヤー1の年で算出。現在の周期を自動ハイライト)
- 上級鑑定:誕生数チャート(生年月日の数字を九宮に配置。強みの矢・最も濃い数・空白の数を読み解き)
- 総合鑑定で「いまの季節×三大周期×今年の運気」の現在地を読み解き
- 「計算の内訳」で文字ごとの換算表と還元過程を表示
- 鑑定書の冒頭に目次ナビ(タップで各セクションへジャンプ。表示される項目に応じて自動生成。印刷時は非表示)

**相性診断**
- ライフパス45通りすべて書き下ろしの相性鑑定(調和度・ご縁のかたち・アドバイス)
- 魂の共鳴(ソウル)/外から見たお二人(パーソナリティ)
- 二人のパーソナルイヤー比較

**共通**
- オフライン対応(Service Worker。一度表示すれば圏外でも鑑定可能。ページは常にネットワーク優先で最新を表示し、画像等はキャッシュ優先+裏で更新)
- 入力内容の記憶(オプトインのチェックボックス。端末内のlocalStorageのみ使用・外部送信なし。チェックを外すと即削除)
- テキスト欄・セレクトでのEnterキーでも鑑定を実行(フォームsubmit対応)
- 鑑定書の印刷・PDF保存(白背景に自動反転。印刷後は「計算の内訳」の開閉状態を自動復元)
- スマートフォン最適化/レスポンシブ対応
- 画面右上のボタンで文字サイズを切替可能(小・標準・大。パソコン・スマートフォン共通。選択は端末内に記憶)
- スマートフォンの「ホーム画面に追加」に対応(`site.webmanifest`。ブランドアートワークのアプリアイコン・Android円形マスク対応版を同梱)
- ナイトモード(紺×金)/ライトモード(生成り×金)を画面右上で切替可能(選択は端末内に記憶。印刷時は従来通り白背景)
- ライフパス計算方式の切替(一括合算方式/桁別還元方式)
- 全角英字・マクロン付きローマ字(Ō・Ū など)の自動補正
- 選択した年・月に合わせて「日」の選択肢を自動調整
- 入力された情報は端末内でのみ計算(外部への送信・保存は一切なし)
- プライバシーポリシー・免責事項ページを同梱(`privacy.html` / `disclaimer.html`。フッターのリンクから閲覧可能。運営:Wind Bell)

## テスト

計算エンジンと鑑定書レンダリングの自動テストを同梱しています(Node.js があれば実行可能。依存パッケージ不要)。

```
node tests/test.js
```

ライフパス両方式・マスターナンバー保持・三大周期・誕生数チャート(矢/空白/最多数)・カルミック検出・12ヶ月ごよみの年またぎ・目次リンク切れ・入力記憶・境界日付・相性診断などを検証します。`index.html` を変更した際は必ず実行してください。

## GitHub Pages での公開手順

1. GitHubで新しいリポジトリを作成(例: `numerology-reading`)
2. このフォルダ内のファイル(`index.html`・`privacy.html`・`disclaimer.html`・`site.webmanifest`・`sw.js`・`images` フォルダ・`README.md`)をアップロードしてコミット
3. リポジトリの **Settings → Pages** を開く
4. **Source** で「Deploy from a branch」を選び、Branch を `main` / `(root)` に設定して Save
5. 数分後、`https://ユーザー名.github.io/リポジトリ名/` で公開されます
6. OGP タグ(SNSシェア用カード)・canonical・構造化データ(JSON-LD)は現在の公開URL `https://numerology.matrix.jp/` 向けに設定済みです。別のURLで公開する場合のみ、`index.html` 内の `canonical`・`og:url`・`og:image` と JSON-LD 内の `url`・`image` を書き換えてください

## 技術メモ

- 依存ライブラリなし(Google Fonts の Shippori Mincho / Cormorant Garamond のみ読み込み)
- 計算方式: ピタゴラス式(A=1〜Z=8)。母音は A・E・I・O・U、Y は子音扱い
- ライフパスは「一括合算方式(生年月日の全桁を合計)」と「桁別還元方式(年・月・日を個別に還元して合算)」を画面上で切替可能(初期値: 一括合算)。数秘術は流派により計算方式が異なり、マスターナンバーの出方が変わることがあるため両対応
- パーソナルイヤーは 1月1日切り替えで算出
- 名前入力はヘボン式ローマ字を想定(全角英字・マクロン付き文字「Ō」などは自動で半角・基本ラテン文字に補正)
- OGP・Twitterカード対応(シェア用画像 `images/og-image.jpg` 同梱。別URLで公開する場合は `canonical`・`og:url`・`og:image`・JSON-LD の書き換えが必要)
- 構造化データ(JSON-LD / `WebApplication`)と `canonical` を `<head>` に設定
- Service Worker(`sw.js`)によるオフライン対応。HTMLはネットワーク優先(常に最新)、画像等はキャッシュ優先+裏で更新。相対パス登録のためサブパス配下(GitHub Pages等)でも動作。`file://` で開いた場合は登録をスキップ
- 入力記憶はオプトイン(`localStorage` キー `numerology-inputs`)。チェックを外すと即削除。プライバシーポリシー第1条・第2条に明記済み
- 文字色はライトモードを含めWCAG AA(コントラスト比4.5:1)を満たすよう調整
- ロゴ・ファビコン・各種アイコンは `images/favicon.png`(正方形アートワーク)、OGP画像は `images/numerology.jpg` から生成し、`images/` に同梱
